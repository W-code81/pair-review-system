Introduction to P.R.S

~By Oluwadamilola and Precious

The Peer Review System (PRS) is a core practice in modern software development, commonly carried out through GitHub pull requests. It allows developers to review each other’s work before merging, helping teams catch errors, follow coding standards, and improve code clarity. Beyond quality control, PRS encourages collaboration, accountability, and shared understanding of the codebase.

Key Prompts and Commands used in P.R.S

Creating Pull Requests

Developers begin by creating a new branch, making changes, and committing them. After pushing the branch to GitHub, a Pull Request (PR) is opened using prompts like “Compare & pull request” and “Create pull request”. The PR description explains what was changed and how it should be reviewed.

Review and Feedback

Reviewers use tools such as comments, approvals, and change requests to give feedback directly on the code. Inline comments make discussions clear and contextual. Authors may then update their work using commands like git pull, git fetch, or git merge to respond to feedback.

Merging Changes

Once all feedback is resolved and checks pass, the PR can be merged into the main branch. GitHub provides merge options such as merge commit, squash, or rebase. After merging, the feature branch is usually deleted to keep the repository clean.

Key Benefits of P.R.S

Improves Code Quality

Peer review helps catch bugs, security issues, and logical errors before code reaches production. Reviewers also suggest cleaner structures and better naming, leading to more readable and maintainable code.

Encourages Collaboration

Reviewing each other’s work helps team members understand different parts of the codebase. This reduces dependency on a single developer and promotes shared ownership, discussion, and consistent coding practices.

Builds Accountability

Every pull request creates a record of changes, feedback, and approvals. This history supports debugging, audits, and future development while helping less experienced developers learn through constructive feedback.
