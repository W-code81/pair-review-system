Pair Review System

A simple demonstration of a pair review workflow using GitHub

Project Overview

This project demonstrates a simple Pair Review System where two participants collaborate on a single webpage. Each participant contributes content and reviews the other person's changes using GitHub Pull Requests.

Contributor A (You)

This section contains the work submitted by Contributor A. It can be a short explanation, assignment, or description of a task.

Contributor B (Partner)

This section contains the work submitted by Contributor B. This content is reviewed through a GitHub Pull Request.

Pair Review Workflow

  1. Contributor B forks the repository from Contributor A.
  2. Contributor B edits their assigned section and pushes changes.
  3. A Pull Request is opened on GitHub.
  4. Contributor A reviews the changes and provides feedback.
  5. Changes are approved and merged into the main branch.
  6. Roles can be swapped and the process repeated.
